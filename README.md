# Домашнее задание №3

## Хранение информации о товарах

Необходимо дополнить домашнее задание №2: написать слой Infrastructure (DAL).
То есть нужно заменить заглушки, которые вы использовали для хранения данных в оперативной памяти на хранение в БД.

## Окружение
Вы можете воспользоваться [этой инструкцией](prepare-enviroment.md) и поднять БД в докере.
Но если возникнут сложности, можете подключиться напрямую к БД, созданной просто локально например в pgAdmin.

## Сущности

```
Товар
{
  Наименование товара;
  Цена товара;
  Вес товара;
  Вид товара{Общий, Бытовая химия, Техника, Продукты};
  Дата создания;
  Номер cклада;
}
```

## Основное задание

* Нужно создать таблицу(цы) путем написания миграции через FluentMigration.
* Необходимо запускать миграции по аналогии с тем, как это было сделано на практическом занятии.
* Нужно реализовать слой Infrastructure:
  * На ручку создания товара необходимо добавить сохранение в БД, а также сразу вернуть Id созданного товара.
  * На ручку получения список товаров с фильтрами и пагинацией необходимо написать select запрос с фильтрами и параметрами.
  * На ручку получения товар по Id необходимо написать простой select запрос.
* Необходимо добавить новую ручку для обновления параметров товара и сделать для нее update запрос к БД.
* Запросы необходимо делать через Dapper.


## Дополнительное задание

- Напишите юнит и интеграционные тесты на эти ручки. Учтите, что в интеграционных тестах необхоидмо проверить в первую очередь слой Infrastructure и ручки в нем.
